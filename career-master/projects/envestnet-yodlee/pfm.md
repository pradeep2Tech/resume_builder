# PFM — Personal Finance Management

**Company:** [Envestnet Yodlee](../companies/envestnet-yodlee.md)  
**Period:** December 2011 — August 2013  
**Role on project:** Team Lead  
**Team size:** 35  
**Clients:** ANZ, RBC, 22Seven, Nedbank, Nimble, Xero, and others

## Problem

Consumers needed a unified view of personal financial information — expenses, spending analysis, and alerts — aggregated from multiple financial institutions via secure web crawling.

## Solution

- Consumer-centric PFM application using Yodlee's patented web-crawler APIs
- API design with advanced patterns: Singleton, Factory, Abstract Factory, Builder, Prototype, Command, Visitor, Observer, Decorator
- Frameworks to minimize code redundancy and improve developer productivity
- Cross-functional team coordination across 35 members
- Product-level recommendations for design and development
- Struts, Spring on JBoss; ANT build

## Impact

- Delivered PFM for tier-1 banking clients (ANZ, RBC, 22Seven, Nedbank, Nimble, Xero)
- Spot Award for 22Seven client appreciation; ANZ appreciations
- Scalable API frameworks reused across Yodlee PFM sub-projects
- Led 35-member team across multiple concurrent client deliveries
