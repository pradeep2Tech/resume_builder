# my Monitor

**Company:** [Envestnet Yodlee](../companies/envestnet-yodlee.md)  
**Period:** November 2014 — January 2015  
**Role on project:** Team Lead / Requirement Analyst  
**Team size:** 2  
**Client:** Yodlee — internal advisory application

## Problem

Customers filing service requests often required Yodlee internal team involvement for data points that could be self-served, slowing resolution and increasing support load.

## Solution

- Internal self-service tool for effective service request filing
- Immediate data points for customers without internal team escalation
- End-to-end design: requirements → feasibility → development → unit testing
- Spring MVC, Jackson, JSON APIs on JBoss
- Design patterns: Singleton, Factory, Visitor, Observer; static rule framework

## Impact

- Improved customer self-service for service request workflows
- Reduced dependency on Yodlee internal teams for routine data access
- Fast delivery with lean 2-person team in 3-month window

## Tech stack

`Java` `Spring MVC` `Jackson` `JSON` `JBoss` `ANT` `Static rule framework`

## Design patterns

Singleton · Factory · Visitor · Observer · Static Rule Framework
